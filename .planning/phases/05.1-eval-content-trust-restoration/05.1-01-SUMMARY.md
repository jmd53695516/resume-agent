## Auto-Fixed Issues (Deviations from Plan)

### Rule 1 — Bug fixes inline

**1. [Rule 1 - Bug] npm eval:reset-rl alias used wrong pattern (`tsx scripts/...`)**
- **Found during:** Task 2 PART C smoke-test
- **Issue:** Bare `tsx scripts/reset-eval-rate-limits.ts` doesn't load `.env.local`; src/lib/env.ts zod-parses at module-init and threw with 11 missing-env errors.
- **Fix:** Mirror the pattern used by the existing `eval` script: `node --env-file-if-exists=.env.local --import tsx scripts/reset-eval-rate-limits.ts`.
- **Commit:** `699c294` (folded into Task 2's atomic commit before publish).

**2. [Rule 1 - Bug] reset-eval-rate-limits.ts cleared 0/N keys against actual @upstash/ratelimit storage**
- **Found during:** Task 3 PART E final verification — the eval CLI was being ratelimited mid-run despite running `npm run eval:reset-rl` first.
- **Issue:** Original script called `redis.del(<bare-prefix>:<id>)` (e.g. `resume-agent:rl:ip10m:::1`). @upstash/ratelimit actually stores under `<prefix>:<id>:<windowTimestamp>` (e.g. `resume-agent:rl:ip10m:::1:2964058`). The bare-prefix delete silently no-op'd.
- **Fix:** Use `redis.keys('<prefix>:<id>:*')` to expand each base into the actual windowed keys; collect into a Set; one variadic redis.del at the end. Blast radius unchanged (explicit prefix list, no globs across unrelated namespaces).
- **Commit:** `4281c3b` (separate fix commit; verified live by post-fix `cleared 3/6 keys`).
