```ts
function extractLastUserText(msgs: UIMessage[]): string | null {
  for (let i = msgs.length - 1; i >= 0; i--) {
    const m = msgs[i];
    if (m.role !== 'user') continue;
    const text = (m.parts ?? []).filter((p: any) => p?.type === 'text').map((p: any) => p.text).join('');
    if (text.trim()) return text;
  }
  return null;
}
```

**Two deliberate adjustments to RESEARCH.md's template the executor MUST make:**

**Adjustment 1: Wrap the classifier deflection's `persistDeflectionTurn` in a try/catch.** RESEARCH.md's code does bare `await persistDeflectionTurn(...)`. If a network blip makes the Supabase insert fail, the user gets a crashed `/api/chat` instead of the deflection — worst-case UX. Wrap every deflection-path `persistDeflectionTurn` call in a try/catch with `console.error`; on failure continue to `return deflectionResponse(...)` anyway. This applies to ALL FIVE deflection sites: turncap, spendcap, ratelimit, borderline, non-normal verdict. Example:

```ts
try {
  await persistDeflectionTurn({ session_id, user_text: lastUser, verdict: null, deflection_text: DEFLECTIONS.turncap, reason: 'turncap' });
} catch (e) {
  console.error('persistDeflectionTurn(turncap) failed', e);
  // Don't fail the response — user still gets the deflection; Phase 4 admin will flag the missing row.
}
log({ event: 'deflect', reason: 'turncap', session_id });
return deflectionResponse('turncap');
```

Justification: D-G-05 says persistence failures should be logged but not propagated. RESEARCH.md does this for the onFinish path (try/catch around persistNormalTurn) but not for the deflection paths. Fix in implementation.

**Adjustment 2: Add `onError` AND `onAbort` handlers for the abort-mid-stream case (Pitfall G).** RESEARCH.md's template has only `onError`. v6's streamText fires `onError` on error and `onAbort` on client-initiated abort. Both cases mean we never got a final assistant text to persist — best-effort record a partial row:

```ts
onError: async (e) => {
  console.error('streamText error', e);
  try {
    await persistDeflectionTurn({
      session_id, user_text: lastUser, verdict,
      deflection_text: '[streamText error]',
      reason: 'offtopic', // closest-fit — this is not really a classifier verdict; use offtopic so downstream admin knows it was a stream failure, not a normal turn
    });
  } catch {}
},
onAbort: async () => {
  // User tab-closed mid-stream. Log only; no row insert (we don't have a useful content yet).
  log({ event: 'chat_aborted', session_id, classifier_verdict: verdict.label });
},
```

**Note on `onAbort` availability:** if TypeScript complains that `onAbort` is not in v6's `StreamTextOptions`, check the v6 changelog; if the name is different (e.g., `onClose`, `onCancel`), use that instead. If no such option exists in v6, leave just `onError` and note this in the task commit message — Phase 4 observability will add abort-tracking through a different mechanism.
