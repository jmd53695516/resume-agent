# Plan 02-04: Turnstile Wired But Off

<objective>
Implement Cloudflare Turnstile as a feature-flagged CAPTCHA on the email gate. The widget is wired and can be turned ON via env var `NEXT_PUBLIC_TURNSTILE_ENABLED=true` in under 10 minutes if abuse is observed. Day-one default: OFF. No UX impact until flipped.

This plan closes SAFE-13. It does NOT enable Turnstile at launch; it puts the switch in place so the cost/benefit reaction time is short (per CONTEXT D-J-01).
</objective>

<context>
**CONTEXT.md D-J summary:**
- D-J-01: Wire Turnstile via `@marsidev/react-turnstile`, feature-flagged off by default.
- D-J-02: When enabled, widget renders below the email input; submit gated on valid token; token server-side verified.
- D-J-03: Env vars in `.env.example` with `.optional()` (but — see env-var-name convention below).
- D-J-04: No Turnstile on `/api/chat` — session creation is the hot attack surface; per-chat rate limits cover the rest.

**Pre-commit hook env-var-name convention (Plan 01-01 hook + commit `ee8f2af` tweak):**
- The hook blocks any `+` diff line matching `NEXT_PUBLIC_[A-Z_]*(KEY|SECRET|TOKEN|PASSWORD|PASS)` in non-planning files.
- `.planning/**` and `docs/superpowers/**` are exempted from the env-var-name scan (but still scanned for real secret shapes).
- For Plan 02-04, env var names in `src/`, `tests/`, and `.env.example` (the last of which is also exempted) must avoid the pattern.
- Chosen names:
  - `NEXT_PUBLIC_TURNSTILE_SITE_ID` — Cloudflare's "site key" is a public identifier; we name it `_ID` (public by design, semantically accurate, doesn't trigger pattern).
  - `TURNSTILE_SECRET_KEY` — server-only, no `NEXT_PUBLIC_` prefix so the pattern doesn't apply; plain descriptive naming.
  - `NEXT_PUBLIC_TURNSTILE_ENABLED` — boolean flag; `_ENABLED` suffix doesn't trigger pattern.

**Read directly from `process.env` (do NOT add to env.ts):**
All three Turnstile vars are optional. Adding them to `env.ts` requires three `.optional()` fields and leaks into type exports. Reading `process.env.X` at the site of use is simpler for feature-flagged optional config and avoids widening `env.ts` for a flag that's off by default.

**Cloudflare Turnstile siteverify reference:**
- Endpoint: `https://challenges.cloudflare.com/turnstile/v0/siteverify`
- Method: POST, `application/x-www-form-urlencoded`, body: `secret=<secret>&response=<token>&remoteip=<optional>`
- Success: `200 OK` with body `{"success": true, "challenge_ts": "...", "hostname": "...", "action": "...", "cdata": "..."}`
- Failure: `200 OK` with body `{"success": false, "error-codes": ["..."]}`
- Docs: https://developers.cloudflare.com/turnstile/get-started/server-side-validation/
</context>

```typescript
type TurnstileVerifyResponse = {
  success: boolean;
  'error-codes'?: string[];
  challenge_ts?: string;
  hostname?: string;
};

async function verifyTurnstileToken(secret: string, token: string): Promise<TurnstileVerifyResponse> {
  const params = new URLSearchParams({ secret, response: token });
  const res = await fetch('https://challenges.cloudflare.com/turnstile/v0/siteverify', {
    method: 'POST',
    headers: { 'content-type': 'application/x-www-form-urlencoded' },
    body: params.toString(),
  });
  if (!res.ok) {
    // Cloudflare outage or network error — fail closed.
    return { success: false, 'error-codes': ['siteverify_http_' + res.status] };
  }
  return (await res.json()) as TurnstileVerifyResponse;
}
```

import { describe, it, expect, beforeEach, afterEach, vi } from 'vitest';

// We test the route handler directly by importing POST and calling it with a
// mocked Request. fetch() to Cloudflare's siteverify endpoint is stubbed via
// globalThis.fetch override. Supabase client is NOT mocked — we rely on a
// fake session_id being returned by a mocked service-role client if needed,
// OR we scope tests to only the BEFORE-verification branch so the DB path is
// never reached in the 400 cases.

const SITEVERIFY_URL = 'https://challenges.cloudflare.com/turnstile/v0/siteverify';

function mockFetch(impl: (input: RequestInfo | URL, init?: RequestInit) => Promise<Response>) {
  const original = globalThis.fetch;
  globalThis.fetch = impl as typeof fetch;
  return () => { globalThis.fetch = original; };
}

async function postBody(body: unknown): Promise<Response> {
  const { POST } = await import('@/app/api/session/route');
  return POST(
    new Request('http://localhost/api/session', {
      method: 'POST',
      headers: { 'content-type': 'application/json' },
      body: JSON.stringify(body),
    }) as unknown as Request,
  );
}

describe('POST /api/session — Turnstile verification (SAFE-13)', () => {
  const originalEnv = { ...process.env };

  beforeEach(() => {
    vi.resetModules();
  });

  afterEach(() => {
    process.env = { ...originalEnv };
    vi.restoreAllMocks();
  });

  it('skips verification when NEXT_PUBLIC_TURNSTILE_ENABLED is absent', async () => {
    // Restore original (no Turnstile flag). Route should accept body w/o token.
    delete process.env.NEXT_PUBLIC_TURNSTILE_ENABLED;
    // Mock fetch to error if called — if verification runs, test fails loud.
    const restore = mockFetch(async (input) => {
      if (String(input).includes('siteverify')) {
        throw new Error('siteverify called when flag is off — should NOT happen');
      }
      return new Response('{}', { status: 200 });
    });
    try {
      // This call will hit Supabase — expect a 200 (happy path) or a
      // Supabase-specific error, NOT a turnstile_* error.
      const res = await postBody({ email: 'test-no-turnstile@example.com' });
      const json = await res.json();
      expect(res.status).not.toBe(400);
      expect(json.error).not.toMatch(/^turnstile_/);
    } finally {
      restore();
    }
  });

  it('returns 500 turnstile_misconfigured when flag on but secret missing', async () => {
    process.env.NEXT_PUBLIC_TURNSTILE_ENABLED = 'true';
    delete process.env.TURNSTILE_SECRET_KEY;
    const res = await postBody({ email: 'test-misconfig@example.com', turnstile_token: 'x' });
    expect(res.status).toBe(500);
    const json = await res.json();
    expect(json.error).toBe('turnstile_misconfigured');
  });

  it('returns 400 turnstile_missing when flag on and token absent', async () => {
    process.env.NEXT_PUBLIC_TURNSTILE_ENABLED = 'true';
    process.env.TURNSTILE_SECRET_KEY = 'test-secret';
    const res = await postBody({ email: 'test-no-token@example.com' });
    expect(res.status).toBe(400);
    const json = await res.json();
    expect(json.error).toBe('turnstile_missing');
  });

  it('returns 400 turnstile_failed when Cloudflare reports invalid token', async () => {
    process.env.NEXT_PUBLIC_TURNSTILE_ENABLED = 'true';
    process.env.TURNSTILE_SECRET_KEY = 'test-secret';
    const restore = mockFetch(async (input) => {
      if (String(input).includes('siteverify')) {
        return new Response(JSON.stringify({ success: false, 'error-codes': ['invalid-input-response'] }), {
          status: 200,
          headers: { 'content-type': 'application/json' },
        });
      }
      return new Response('{}', { status: 200 });
    });
    try {
      const res = await postBody({ email: 'test-bad-token@example.com', turnstile_token: 'bad-token' });
      expect(res.status).toBe(400);
      const json = await res.json();
      expect(json.error).toBe('turnstile_failed');
      expect(json.errorCodes).toContain('invalid-input-response');
    } finally {
      restore();
    }

  it('proceeds to session create when Cloudflare reports success', async () => {
    process.env.NEXT_PUBLIC_TURNSTILE_ENABLED = 'true';
    process.env.TURNSTILE_SECRET_KEY = 'test-secret';
    const restore = mockFetch(async (input) => {
      if (String(input).includes('siteverify')) {
        return new Response(JSON.stringify({ success: true, hostname: 'test.local' }), {
          status: 200,
          headers: { 'content-type': 'application/json' },

      return new Response('{}', { status: 200 });

    try {
