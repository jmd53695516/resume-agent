## Decisions Made

- **Recovery from mid-plan session drop**: Task 1 committed cleanly in a prior session (0805622) and Task 2 code was on disk dirty but uncommitted when the session network-errored. Resume strategy: (1) inspect the dirty diff against the plan's 13 grep acceptance criteria — all pass on the disk state, (2) re-run the verification gates (tsc + build + tests) — all green, (3) commit Task 2 atomically via a single git add + commit (f6e546c), (4) write this SUMMARY + commit metadata. Zero code rework needed.
- **Inner `if (view === 'matrix')` gates cleanup registration**: the plan's exact pattern at lines 262-267 honored verbatim. The alternative (`useEffect(() => { document.body.classList.toggle('matrix-mode', view === 'matrix'); return () => document.body.classList.remove('matrix-mode'); }, [view])`) would noop-remove the class on every chat→matrix→chat cycle. Cheaper to gate the cleanup registration and avoid the noop.
- **Literal initial state `useState<'chat' | 'matrix'>('chat')`**: D-B-03's no-persistence rule isn't just a UX choice — it's also a hydration-safety guarantee. A lazy initializer reading from `sessionStorage.getItem('view')` would diverge between server prerender ('chat' fallback) and client first-render (whatever was stored) → React error #418/#423 in production. The hydration safety IS the design medium.
- **ViewToggle as sibling of ChatUI, not child**: the pill is `position: fixed top-[18px] right-[18px] z-50` and absolutely should not be a child of ChatUI's `<main>` flex column. Sibling mount inside the page-level Fragment also keeps the state-lift symmetric — page owns view, page owns both the writer (ViewToggle.onChange) and the reader (future Plan 05 will pass `view` to ChatUI for the canvas mount).
- **ChatUI prop surface UNCHANGED**: still receives only `sessionId`. Plan 05 will add the `view` prop. This plan deliberately avoids touching ChatUI to keep the diff surgical — the body-class CSS cascade is sufficient for everything except the canvas mount (Plan 05's territory).
