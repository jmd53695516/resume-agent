## Spacing Scale

The bundle is **NOT 8-point pure** ‚Äî it uses messaging-app micro-spacing on the bubble interior (8px / 13px / 9px asymmetric padding). Document this exception explicitly so the checker doesn't fail it.

Declared values:

| Token | Value | Usage |
|-------|-------|-------|
| 1 | 1px | `bubble-row` vertical padding |
| 2 | 2px | `chat-inner` gap between bubble rows |
| xs | 4px | Inline gap (gap-1 in header center stack), `chat-inner` left/right padding |
| sm | 8px | Composer gap, plus-btn padding, header right padding |
| md | 12px | Composer left/right padding, `chat-inner` left/right (12px) |
| md+ | 14px | Composer top padding, `chat-inner` top padding, header left padding (14px) |
| lg | 16px | Timestamp top padding, view-toggle button horizontal padding |
| xl | 18px | View-toggle fixed offset (top: 18px, right: 18px), `chat-inner` bottom padding |
| 2xl | 32px | Page outer padding (`body` 32px 16px) |

**Spacing exceptions (deliberate, source-cited):**

- **Bubble interior padding `8px 13px 9px`** ‚Äî asymmetric on purpose (top 8 / sides 13 / bottom 9 is the iMessage-pattern "letter sits on the baseline" feel). Source: `Chat Stream.html:165`. DO NOT round to 8/12 ‚Äî the asymmetry is the visual signature.
- **Header height fixed `88px`** ‚Äî designed to seat the 38px avatar + 11px contact name + 14/8/10 padding. Source: `Chat Stream.html:84-87`.
- **Composer input shell height `34px` with radius `17px`** ‚Äî pill shape (radius = height/2). Source: `Chat Stream.html:215-217`.
- **Toggle pill outer padding `3px`** ‚Äî tight collar around the two pill-buttons; gives the active pill its own visual island. Source: `Chat Stream.html:244`.

## Grouping Rules (D-A-01)

A **run** is consecutive messages from the same sender (collapsed `m.role === 'user'` vs `m.role === 'assistant'`; `system` messages are skipped). Position values per message:

- `'only'` ‚Äî single-message run (bubble before AND after are different sender, OR no neighbor)
- `'first'` ‚Äî first message in a multi-message run (bubble after is same sender)
- `'middle'` ‚Äî interior of a 3+ message run (bubbles before AND after are same sender)
- `'last'` ‚Äî last message in a multi-message run (bubble before is same sender)

**Helper:** `computePositions(messages: {id, role}[]): Map<id, position>` ‚Äî single pass, O(n). See RESEARCH ¬ßPattern 2 for the implementation. Living at `src/lib/chat-format.ts` (NEW).

### Radius math (per `chat.jsx:90-106`, verified in bundle)

Default all corners = `20px`. Then squash the inside corner(s) where the run continues to `6px`:

| Position | User (right-aligned, `--me`) | Assistant (left-aligned, `--them`) |
|----------|------------------------------|-------------------------------------|
| `only` | TL 20 / TR 20 / BL 20 / BR 20 | TL 20 / TR 20 / BL 20 / BR 20 |
| `first` | TL 20 / TR 20 / BL 20 / **BR 6** | TL 20 / TR 20 / **BL 6** / BR 20 |
| `middle` | TL 20 / **TR 6** / BL 20 / **BR 6** | **TL 6** / TR 20 / **BL 6** / BR 20 |
| `last` | TL 20 / **TR 6** / BL 20 / BR 20 | **TL 6** / TR 20 / BL 20 / BR 20 |

**Important ‚Äî radius applied via inline style** (`style={{ borderTopLeftRadius: 20, ... }}`), NOT via Tailwind classes. Reason per RESEARCH Pattern 3: matrix-mode CSS does NOT need to override radii (bundle confirms only bg/color/border/box-shadow change in matrix mode), so inline-style + class-based color overrides coexist cleanly with no `!important` battle.

**Grouping does NOT break across timestamps** (RESEARCH Open Q2 resolved): same-sender messages straddling a timestamp divider are still ONE run. Bundle's `groupMessages` is sender-only. Same-sender + >5min gap ‚Üí one bubble run with squared interior corners AND a timestamp divider in between. If this looks weird in real usage, "break run on timestamp" is a 3-line change in `computePositions` ‚Äî defer until observed.

**Grouping for the in-flight typing indicator:** the `thinking-indicator` (shown during `status === 'submitted'`) is NOT part of `messages` and is rendered AFTER the messages map. It does NOT participate in grouping; it's an ephemeral UI hint, not a message. Existing behavior ‚Äî no change.

**Grouping with mixed assistant content (text + MetricCard + TracePanel):** the position prop applies ONLY to the text bubble's radius. Chips and TracePanels keep their own visual styling (rounded-full chip, rounded-rectangle trace). Document this in `MessageBubble.tsx` block comment. (RESEARCH Open Q3 resolved.)

## Timestamp Rendering Rules (D-A-02)

### When to show a timestamp divider

Walk `messages` (filtered to `user` + `assistant` only); for index `i`:

```
shouldShowTimestampBefore(messages, i) =
  i === 0
  OR (messages[i].metadata.createdAt - messages[i-1].metadata.createdAt) > 5 * 60 * 1000 ms
```

If either `createdAt` is missing (legacy / pre-metadata messages), return `false` ‚Äî render nothing rather than `Invalid Date`. Defensive.

### Format

`format(new Date(createdAt), 'h:mm a')` via `date-fns@4.x` ‚Üí `"10:42 AM"` (uppercase AM/PM, single-digit hour, no leading zero).

### Source of `createdAt` (RESEARCH ¬ßPattern 1)

- **User messages:** stamped CLIENT-side at `sendMessage({ text, metadata: { createdAt: Date.now() } })`. The current `ChatUI.tsx:69` `sendMessage({ text: input })` adds a metadata field.
- **Assistant messages:** stamped SERVER-side at `/api/chat/route.ts` via `result.toUIMessageStreamResponse({ messageMetadata: ({ part }) => part.type === 'start' ? { createdAt: Date.now() } : undefined })`. Additive only ‚Äî does NOT change six-gate flow, classifier, tools, or persisted DB shape. **VERIFIED** against `ai@^6.0.168` `index.d.ts:2147-2149, 2273-2276`.

### TimestampDivider component (NEW)

Lives at `src/components/TimestampDivider.tsx`. Renders:

```tsx
<div data-testid="timestamp-divider" className="...">
  {format(new Date(createdAt), 'h:mm a')}
</div>
```

Styling per dimensions table above. Returns `null` if `createdAt` is missing.

### Tunables (port verbatim ‚Äî these are the visual signature)

| Constant | Value | Notes |
|----------|-------|-------|
| `fontSize` | `18px` | Column width = font size |
| `trailFade` | `0.06` | Per-frame black-overlay alpha (lower = longer trails) |
| `flipChance` | `0.04` | Probability a non-head glyph re-randomizes per frame |
| `speedMin` | `0.35` rows/frame | Min column descent speed |
| `speedMax` | `0.95` rows/frame | Max column descent speed |
| `resetTopMin` | `-50` rows | When a column resets, start somewhere above the top |
| `resetTopMax` | `-5` rows | (range for reset start) |
| `resetChanceAtBottom` | `0.018` per frame | Chance per frame a column resets after passing viewport |
| `headWhiteChance` | `0.72` | Probability the head glyph is rendered as `#d6ffe6` (near-white-green); else `#9bff9b` (bright green) |

### Column count (derived)

`Math.ceil(canvasWidth / fontSize)` columns at any viewport width. Resize listener rebuilds columns on window resize.

### Color palette

| Element | Color | Notes |
|---------|-------|-------|
| Head glyph (white variant) | `#d6ffe6` | 72% probability |
| Head glyph (green variant) | `#9bff9b` | 28% probability |
| Head glow | `shadowColor: '#7cff7c'`, `shadowBlur: 8` | Per-frame canvas shadow on head |
| Trail | gradient computed per row index `k`: `rgb(${10*(1-t)}, ${255*(1-t*0.85)}, ${60*(1-t*0.9)})` where `t = k/length`. Bright green `~#00ff66` near head, fading to `~#003311` at tail | Computed inline; no token |
| Trail glow | `shadowBlur: 0` (only head glows) | |
| Page bg behind canvas | `#000` | Stage container bg |

### Per-frame fade

```js
ctx.fillStyle = `rgba(0, 0, 0, ${RAIN.trailFade})`;  // 0.06 alpha
ctx.fillRect(0, 0, width, height);
```

This produces the classic decay-trail effect. DPR scaling: `Math.min(window.devicePixelRatio || 1, 2)` (cap at 2√ó to avoid excess fillrate on retina).

### Frame rate

`requestAnimationFrame` loop = browser-native 60fps target on desktop. No throttling logic.

### Reduced motion + viewport gate (CD-03)

```ts
useEffect(() => {
  if (!visible) return;
  const reduce = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  if (reduce) return;  // canvas mounts but no RAF loop ‚Äî green static
  if (window.innerWidth < 768) return;  // mobile gets no rain
  // ... begin RAF
}, [visible]);
```

### Lazy mount (CD-02)

In `ChatUI.tsx`:

```ts
import dynamic from 'next/dynamic';

const MatrixRain = dynamic(
  () => import('./MatrixRain').then((m) => ({ default: m.MatrixRain })),
  { ssr: false, loading: () => null },
);

// Inside render:
{view === 'matrix' && <MatrixRain visible />}
```

Conditional mount keeps the canvas, RAF loop, and ~5 KB of MatrixRain code out of the default chat-mode bundle. `next/dynamic({ ssr: false })` is valid in client components only ‚Äî `ChatUI.tsx` already has `'use client'` (Pitfall 4 in RESEARCH).

## Spacing / Typography / Color Summary (for Checker)

- **Spacing:** mostly multiples of 4 (16, 32, etc.); deliberate exceptions documented above for bubble interior (8/13/9), header height (88), input shell (34/17), toggle outer pad (3px), toggle button pad (7/16), timestamp pad (16/0/10/0). Asymmetric padding is the iMessage signature, not noise.
- **Typography (chat mode):** 4 sizes ‚Äî 11px (caption/label), 12.5px (toggle), 15px (placeholder/starter), 16px (body). 2 weights ‚Äî 400 (regular) + 500 (label) primary; 600 used on avatar only (acceptable since avatar text is a one-off).
- **Typography (matrix mode):** 4 sizes ‚Äî 10px, 11px, 13px, 14px. 1 weight ‚Äî 400 (Share Tech Mono ships only one weight). All text uppercase except bubble body.
- **Color (chat mode 60/30/10):** 60% bg-page tan (`#f3f1ec` / `#e6e2db` gradient), 30% panel white (`#ffffff`) + dark-grey assistant bubble (`#2c2c2e`), 10% accent blue user bubble (`#2080ff`) + send button. Active toggle uses black-on-white (foreground/panel) ‚Äî NOT the accent blue (acceptable; it's a semantic "active" state, not a CTA).
- **Color (matrix mode):** entire palette is the easter egg. Everything green-on-black is accent.
- **Destructive:** `#d70015` for `chat-error` only. No new destructive surfaces this phase.
