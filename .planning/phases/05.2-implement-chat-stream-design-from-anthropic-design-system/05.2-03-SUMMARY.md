## Decisions Made

- **Recovery path**: prior executor's three task commits landed cleanly before a network error; on resume, inspected each diff against the 30+ grep acceptance criteria in the plan, confirmed all pass, ran mandatory verification gates (tsc + build + tests), and proceeded straight to close-out. No rework was needed — the committed code matches the plan exactly.
- **Assistant timestamp source-of-truth = client React state, not `m.metadata`**: AI SDK v6 sets `UIMessage.metadata` server-side at message-creation via the `messageMetadata` callback; the client cannot mutate it post-hoc. D-A-02-AMENDED forbids `/api/chat/route.ts` modifications, so the only place to record the first-chunk timestamp is in client state (`assistantTimestamps: Record<id, epochMs>`). The render-side `metaView` adapter unifies user (`metadata.createdAt`) + assistant (`assistantTimestamps[id]`) into the shape `shouldShowTimestampBefore` consumes.
- **useEffect dependency on `assistantTimestamps`**: the deps `[status, messages, assistantTimestamps]` cause the effect to re-run after every stamp, but the early-return guard `if (assistantTimestamps[latest.id]) return` short-circuits before the setter fires. No infinite loop. React's same-reference state-setter bailout also kicks in for the duplicate update. Confirmed by 562/562 vitest pass + `next build` strict-mode compile.
- **IIFE render-block over a useMemo hook**: at CHAT-10's 30-turn cap, a one-shot IIFE that runs once per render is trivially cheaper than the useMemo + dep-array bookkeeping. Plan explicitly forbade memoization (Plan 01 RESEARCH line 330).
- **Selector-hook testids placed on EXISTING chrome elements**: no DOM restructure required. The chev SVG was the only NEW DOM node added — wrapped in an `inline-flex items-center gap-[3px]` container alongside the contact-name span so the matrix-mode CSS can grab both via `chat-contact-name` and `chat-contact-chev` testids while the chat-mode CSS leaves the chev grey.
- **Two-step `m.parts as unknown as AssistantProps['parts']` cast preserved**: typed `useChat<ResumeAgentUIMessage>` narrows `metadata` but not `parts` — UIMessage.parts is still wider than MessageBubble's `(TextPart | ToolPart)[]` union. The `unknown` bridge is honest about the narrowing and matches the Phase 03-03 pattern.
