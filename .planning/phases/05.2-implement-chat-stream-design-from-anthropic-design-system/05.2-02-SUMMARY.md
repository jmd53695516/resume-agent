## Decisions Made

- **Append-only edit pattern**: The entire matrix-mode block lives at the END of `globals.css` (start line 234). No inserts inside `@theme inline`, `:root`, `.dark`, `@layer base`, or any pre-existing rule. Verified by `git diff` showing zero `-` lines and a single `@@ -233,0 +234,214 @@` hunk header.
- **`:root` and `.dark` blocks byte-identical**: Confirmed via `git diff` — zero modifications to lines 70-117 (`:root`) or lines 119-163 (`.dark`). `.dark` remains DEAD CODE per D-A-03 (preserved for future true light/dark mode; never activated by the toggle pill).
- **All selectors use `body.matrix-mode` prefix per RESEARCH §Pattern 4** — grep for bare `.matrix-mode` (no `body.` prefix) returns zero matches. This prevents future-CSS-author confusion about scope: matrix-mode is a body-level state, not a component-level modifier.
- **`msg-assistant > div:first-child` (not `> div`)** — the assistant bubble is followed by a sibling chevron div that toggles the TracePanel; styling all children would put the glow on the chevron affordance too. First-child targets just the bubble.
- **Selectors for components not yet built (chat-main, chat-header, chat-avatar, chat-contact-name, chat-contact-chev, chat-composer, timestamp-divider, view-toggle) all live in THIS plan** even though Plans 03/04 own those components. Rationale: keep all matrix-mode CSS in one logical block; downstream plans just add the matching `data-testid` attributes rather than scatter matrix CSS across N files.
