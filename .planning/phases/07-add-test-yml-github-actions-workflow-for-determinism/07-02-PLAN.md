## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR head SHA → workflow execution | Untrusted PR code runs in GH Actions with the workflow's permissions; commit-SHA pin (not branch ref) prevents tag-swap attacks |
| `secrets.*` → workflow env | Phase 7 D-B-02..04 mandates ZERO real secrets; structural separation from eval.yml spend path |
| `main` branch protection rule → merge gate | A misconfigured rule could make main unmergeable (D-C-04 risk) or fail-open (no gate at all) |
| GH Actions runner → public repo | Public repo means anyone can open a PR; workflow runs on untrusted code with `pull_request` (not `_target`) — runner has NO write tokens |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
