<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local dev shell → committed config | Vitest/package.json changes affect every future CI run and every local-dev run for every contributor |
| `process.env.CI` → test discovery | A misconfigured CI env (or stale CI=1 in a local shell) silently changes which tests run |
| Lint surface narrowing | `eslint src/` skips tests/scripts/evals — type-of-error-class shifts but no security boundary loss because tests/scripts/evals don't run in prod |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-07-01-02 | Repudiation | Future contributor removes the CI-gated exclude without filing a follow-up | accept | Inline TODO comment references `deferred-items.md`; removal would surface in diff review |
