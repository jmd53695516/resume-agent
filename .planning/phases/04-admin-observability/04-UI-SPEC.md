# Phase 4 — UI Design Contract

> Visual and interaction contract for the admin dashboard pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase INHERITS the design system established in Phases 1–3 and EXTENDS it with admin-surface tokens. Nothing here contradicts globals.css.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (vendored in `components/ui/`) |
| Preset | `base-nova`, `neutral`, cssVariables (from `components.json`) |
| Component library | Radix UI (via shadcn) |
| Icon library | `lucide-react` (already installed; `ChevronRight`, `ChevronDown` in use) |

**shadcn gate result:** `components.json` already exists. Design system is initialized. No re-initialization needed.

## Spacing Scale

Inherits Tailwind v4 defaults (multiples of 4). Phase 4 uses this subset:

| Token | Value | Admin Usage |
|-------|-------|-------------|
| xs | 4px | Badge padding, icon gaps in table cells |
| sm | 8px | Table cell vertical padding (`py-2`), nav item gaps |
| md | 16px | Card padding, section spacing, nav horizontal padding |
| lg | 24px | Page-level horizontal gutter (`px-6`), card grid gap |
| xl | 32px | Page top margin, section header bottom margin |
| 2xl | 48px | Major section breaks on `/admin/health` |
| 3xl | 64px | Not used in admin views |

Exceptions:
- Top-bar nav height: 44px (touch-target minimum; explicit `h-11`)
- Top-bar nav active indicator: 2px underline (`border-b-2`)
- Abuse log row: single-line `py-1` (data-density; not standard `py-2`)
- Session header summary: `py-3` to give the session metadata visual breathing room

## Typography

Inherits the font stack from Phase 2/3 (`globals.css`). Admin-surface hierarchy:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Table cell content, log rows, badge labels |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.4 | Column headers (`uppercase tracking-wide`), card section labels, nav links |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 | Page titles (`/admin/sessions`, `/admin/cost`, etc.) |
| Display | 28px (`text-2xl`) | 600 (semibold) | 1.15 | Cost card dollar amounts (the big number on each cost card) |

Notes:
- 2 weights — 400 regular + 600 semibold
- Admin body is 14px (smaller than public chat's 16px) to support data-density over polish
- Mono font (`font-mono`, `--font-mono` = Geist Mono) used exclusively in: abuse log `ip_hash` truncation, trace panel JSON, `created_at` timestamps in compact contexts
- All timestamps displayed in Joe's local timezone via browser-side `toLocaleString()` — use a client component wrapper `<LocalTime iso={string} />` for SSR-safe rendering

## Color

Inherits all tokens from `globals.css`. Admin surfaces use these roles:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg-page` (`#f3f1ec` light / `#1a1a1c` dark) | Admin page background — same as chat |
| Secondary (30%) | `--panel` (`#ffffff` light / `#000000` dark) | Cards, top-bar, table surface |
| **Admin chrome** | `--muted` (`rgba(120,120,128,0.16)`) | **Top-bar nav background** — subdued grey, not brand color. No `--me` (blue) in any admin chrome. |
| Accent (10%) | `--me` (`#2080ff`) | Active nav underline only; "View transcript" link in session notification email |
| Destructive | `--destructive` (`#d70015` light / `#ff453a` dark) | `output-error` trace states (inherited from TracePanel); 403 / NotAuthorized page warning text |

**Status colors (inherited from StatusBanner, Phase 3):**

| Semantic | Token | Tailwind class | Usage |
|----------|-------|---------------|-------|
| ok / green | `bg-green-100 text-green-800` | `bg-green-100 text-green-800` | Per-dep "ok" badge on `/admin/health` |
| degraded / yellow | `bg-amber-100 text-amber-900` | `bg-amber-100 text-amber-900` | Per-dep "degraded" badge; same class as `StatusBanner` |
| down / red | `bg-red-100 text-red-800` | `bg-red-100 text-red-800` | Per-dep "down" badge on `/admin/health` |

**Top-bar nav specifics:**
- Background: `bg-muted` (the `rgba(120,120,128,0.16)` muted token over `--panel`) produces the subdued grey admin look
- Active nav item: `font-semibold border-b-2 border-[--me]` (underline in accent blue + bold text)
- Inactive nav item: `text-muted-foreground hover:text-foreground` — no underline
- Accent (`--me` blue) is ONLY used for the active-state underline in the top-bar; nowhere else in admin chrome

**Priority badge color (sessions table):**
- Company-domain rows: `bg-blue-50 text-blue-700 border border-blue-200` — small badge labeled `PRIORITY`
- Flagged classifier rows: `bg-amber-100 text-amber-800` — labeled `flagged`
- Combined (priority + flagged): show both badges stacked/inline

## Page-by-Page Interaction Contract

### 1. `/admin/login`

**Purpose:** Single sign-in gate. Only unauthed-accessible admin route.

**Layout:** Centered vertically and horizontally. Full viewport height. White card (`--panel`) centered on `--bg-page` background.

**Elements:**
- Heading: "Admin sign in" — `text-xl font-semibold` — centered
- Body text: "Sign in with your GitHub account to access the dashboard." — `text-sm text-muted-foreground` — centered
- CTA button: shadcn `<Button size="lg">` with GitHub icon (lucide `Github`) — "Sign in with GitHub" — `variant="default"` (uses `--me` blue fill)
- On click: calls `supabase.auth.signInWithOAuth({ provider: 'github', options: { redirectTo: origin + '/auth/callback' } })` — no page transition needed, browser follows OAuth redirect
- Error state (from URL param `?error=oauth_failed`): inline `text-sm text-destructive` message below button — "Sign in failed. Try again or contact Joe."

**No top-bar nav.** This page is outside the admin layout.

### 3. `/admin/sessions` (DEFAULT LANDING)

**Purpose:** Last 100 sessions overview. Joe's primary operational view.

**Layout:** Full-width page within admin layout. Page title "Sessions" at `text-xl font-semibold` with `mb-4`. Table spans full content width.

**Table columns (in order):**

| Column | Data | Notes |
|--------|------|-------|
| Email | `email` | Full address; truncate at 32 chars with `truncate max-w-[200px]` |
| Domain | `email_domain` | Plain string |
| When | `created_at` | Relative — "5 min ago", "2 hours ago" (client-rendered via `<RelativeTime>` component) |
| Flags | `flagged` boolean + company-domain priority | `PRIORITY` badge if non-free-mail domain; `flagged` badge if classifier flagged; both can appear |
| Cost | `total_cost_cents` | Rendered as `$0.14`; right-aligned |
| Turns | `turn_count` | Integer; right-aligned |

**Sorting:**
- Default: `created_at DESC` (newest first)
- Clickable column headers: "When" and "Domain" — push `?sort=created_at&dir=asc/desc` or `?sort=domain&dir=asc/desc` via `<Link>`. No client state needed.
- Active sort column header: `font-semibold` (no arrow icons in Phase 4 — URL param is the truth; header text alone indicates active sort)

**Row interaction:**
- Entire row is clickable — wraps in a `<Link href="/admin/sessions/{id}">` via row `onClick` or `asChild` pattern
- Hover state: `hover:bg-muted/50` row highlight
- Cursor: `cursor-pointer`

**Empty state:**
- Heading: "No sessions yet"
- Body: "Recruiter sessions will appear here once the first chat begins."
- No illustration, no icon — text only, `text-muted-foreground text-sm`

**shadcn components:** `Table`, `TableHeader`, `TableRow`, `TableHead`, `TableBody`, `TableCell`, `Badge`

### 4. `/admin/sessions/[id]` (transcript viewer)

**Purpose:** Full conversation audit with always-expanded tool traces.

**Layout:** Two-zone layout:
- **Session header** (top, full-width card): email, email_domain, ip_hash (first 8 chars), turn count, total cost (dollars), flagged status, `created_at` (full timestamp in local timezone). Uses shadcn `Card` with `p-4 mb-6`.
- **Transcript scroll area** (below header): renders all messages in created_at order.

**Session header fields:**
```
joe@acmecorp.com · acmecorp.com · a3f8b2c1 · 7 turns · $0.42 · [PRIORITY] · May 6, 2026 at 9:14 AM
```
All on one line, `text-sm`, separated by ` · ` (middle-dot), `text-muted-foreground` for secondary fields.

**Message rendering — admin variant:**

Reuses `<MessageBubble>` from Phase 3. Admin variant passes `alwaysExpandTrace` prop:
- `<MessageBubble role="user" text={...} />` — unchanged from Phase 3 (blue bubble, right-aligned)
- `<MessageBubble role="assistant" parts={...} alwaysExpandTrace={true} />` — same bubble visual, but TracePanel renders with `open` forced to `true` (no chevron click; `<details open>` with `pointer-events-none` on the `<summary>`)

**`alwaysExpandTrace` implementation:**
- Pass `alwaysExpanded?: boolean` prop to `<TracePanel>`
- When `alwaysExpanded={true}`: render `<details open>` without `onToggle`; hide the summary chevron (`hidden`); show a static label "Tool trace" instead of "See what I did" to distinguish from public chat context
- `<MetricCard>` renders inline unchanged (no variant needed)

**No streaming in this view.** All data is fully persisted. No `bubble-pop` animation needed (static transcript).

**Empty state:** "This session has no messages." — shown if `messages.length === 0`

### 5. `/admin/cost`

**Purpose:** Rolling cost visibility for 24h, 7d, 30d windows.

**Layout:** Page title "Cost" at `text-xl font-semibold mb-6`. Three shadcn `Card` components in a responsive grid: `grid grid-cols-1 md:grid-cols-3 gap-4`.

**Each cost card (`<CostCard window="24h|7d|30d" data={...} />`):**

```
[Window label]        ← text-xs font-semibold uppercase tracking-wide text-muted-foreground
$14.32                ← text-2xl font-semibold (Display role — the big number)
438 requests          ← text-sm text-muted-foreground

─── Per-tool breakdown ───
  research_company    $8.12  (58%)
  get_case_study      $3.44  (25%)
  design_metric…      $1.90  (14%)
  (no tool)           $0.86   (6%)   ← chat turns without tools

Cache hit rate: 87.4% ← text-sm font-medium, green if ≥80%, amber if 60–79%, red if <60%
```

**Cache hit rate color thresholds (Claude's discretion):**
- ≥80%: `text-green-700` — healthy
- 60–79%: `text-amber-700` — monitor
- <60%: `text-red-700` — investigate (prompt cache may not be warming)

**Window label copy:** "Last 24 hours" / "Last 7 days" / "Last 30 days"

**$0.00 state (no spend):** Show `$0.00` with `text-muted-foreground` on the big number + "No requests in this window." below the breakdown section.

### 6. `/admin/abuse`

**Purpose:** Classifier-flagged messages and rate-limit deflections.

**Layout:** Page title "Abuse log" at `text-xl font-semibold mb-4`. Single full-width list below.

**Row format (one-line, compact):**

```
<time> · <email_user>@<domain> · <ip[:8]> · <verdict_or_reason> · <message[:100]>
```

Example:
```
May 6, 9:02 AM · jane@example.com · a3f8b2c1 · injection · "Ignore all previous instructions and..."
```

**Implementation:** `<ul>` of `<li>` rows. Each row:
- `font-mono text-xs` for ip_hash portion
- `text-sm` for rest of content
- `py-1 border-b border-border/30` (compact density)
- Link to `/admin/sessions/{session_id}` on the row (full row clickable; `hover:bg-muted/40 cursor-pointer`)
- Verdict/reason styled: `text-amber-700 font-medium` for classifier verdicts, `text-red-700 font-medium` for ratelimit deflections

**Verdict display mapping:**
| `classifier_verdict` / `stop_reason` | Display label |
|--------------------------------------|--------------|
| `injection` | `injection` |
| `offtopic` | `off-topic` |
| `sensitive` | `sensitive` |
| `deflection:ratelimit` | `rate limit` |
| `deflection:spendc ap` | `spend cap` |
| other | raw value |

**Empty state:**
- Heading: "No flagged activity"
- Body: "Classifier flags and rate-limit hits appear here."

**No pagination in Phase 4** — last 100 rows, `ORDER BY created_at DESC`. If > 100 rows exist, a footer note: "Showing last 100 flagged events."

### 7. `/admin/health`

**Purpose:** Dependency status grid + heartbeat ages + last 5 alarms + BetterStack link.

**Layout:** Page title "Health" at `text-xl font-semibold mb-6`.

**Section 1 — Dependency grid:**

5 cards in a `grid grid-cols-2 md:grid-cols-5 gap-3`. Each dep card:

```
[Dep name]     ← text-xs font-semibold uppercase tracking-wide text-muted-foreground
[status badge] ← bg-green-100/amber-100/red-100 + matching text color
```

Status badge copy: "ok" / "degraded" / "down" — same tokens as `StatusBanner` (`bg-amber-100 text-amber-900` for degraded, `bg-green-100 text-green-800` for ok, `bg-red-100 text-red-800` for down).

Deps: `anthropic`, `classifier`, `supabase`, `upstash`, `exa`

**Section 2 — Heartbeat ages:**

Below the dep grid. Two-column layout:

| Heartbeat | Age |
|-----------|-----|
| Anthropic cache | "23 seconds ago" |
| Classifier | "41 seconds ago" |

If heartbeat key absent (never fired or expired): show `—` with `text-muted-foreground`.

**Section 3 — Last successful turn:**

"Last successful chat turn: [relative time]" — `text-sm text-muted-foreground`. Sources from `messages WHERE role='assistant' AND stop_reason NOT LIKE 'deflection:%' ORDER BY created_at DESC LIMIT 1`.

If none: "No turns recorded yet."

**Section 4 — Last 5 alarms:**

Heading: "Recent alarms" at `text-sm font-semibold mb-2`.

List: `<ul>` of last 5 `alarms_fired` rows ordered `fired_at DESC`:

```
[condition label] · [fired_at in local time]
```

Condition labels:
| `condition` value | Display |
|-------------------|---------|
| `spend-cap` | Spend cap tripped |
| `error-rate` | Error rate spike |
| `dep-down` | Dependency down |
| `rate-limit-abuse` | Rate limit abuse |

Each row: `text-sm py-1`. Condition label: `font-medium text-amber-700`.

If no alarms: "No alarms fired." — `text-sm text-muted-foreground`.

**Section 5 — BetterStack link:**

Conditional — only shown if `BETTERSTACK_DASHBOARD_URL` env var is set.

```
<a href={env.BETTERSTACK_DASHBOARD_URL} target="_blank" rel="noopener noreferrer"
   class="text-sm text-[--me] underline underline-offset-2 hover:opacity-80">
  View BetterStack status page
</a>
```

No iframe. Link-out only (per RESEARCH §10 — iframe support not confirmed).

### 8. `/admin/layout.tsx` — Shared Admin Shell

**Top-bar nav:**

- Background: `bg-muted` — the subdued grey admin look (not white, not brand blue)
- Height: `h-11` (44px)
- Full-width horizontal flex: `flex items-center px-6 gap-6 border-b border-border`
- Left group: nav links — Sessions / Cost / Abuse / Health (in that order)
- Right group: Refresh button + Sign out button

**Nav link spec:**
- Inactive: `text-sm text-muted-foreground hover:text-foreground transition-colors`
- Active (current page): `text-sm font-semibold text-foreground border-b-2 border-[--me]` — underline sits at the bottom of the nav bar (achieved via `pb-[calc(0.5rem+2px)] mb-[-2px]` or via `border-b-2` on the link itself flush to nav bottom)
- Active detection: compare `pathname` from `usePathname()` (client component `<AdminNav>`)

**Refresh button:**
- `<Button variant="ghost" size="sm">` with `RefreshCw` icon (lucide, 14px) + label "Refresh"
- On click: calls `router.refresh()` from `useRouter()`
- No loading state in Phase 4 (SSR refresh is fast enough at this volume)

**Sign out button:**
- `<Button variant="ghost" size="sm">` — label "Sign out"
- On click: calls `supabase.auth.signOut()` then `router.push('/')`

### 9. `<NotAuthorized />` component

**Purpose:** Shown when `requireAdmin()` returns null for an authenticated-but-not-allowlisted user.

**Layout:** Centered on page (same centering as `/admin/login`).

**Elements:**
- Heading: "Access denied" — `text-xl font-semibold`
- Body: "Your GitHub account is not on the admin allowlist." — `text-sm text-muted-foreground`
- Secondary body: "If you believe this is an error, contact Joe." — `text-sm text-muted-foreground`
- Link: "Go to homepage" — `<Link href="/">` styled as `text-sm text-[--me] underline`
- No shadcn Button here — plain anchor link only

**This component also calls `supabase.auth.signOut()` on mount** (client-side effect) to clear the session cookie, so the user is fully signed out.
