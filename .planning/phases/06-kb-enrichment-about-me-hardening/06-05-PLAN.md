<objective>
Wave 3, Plan 5 of 6. Expand `evals/cat-01-fabrication.yaml` `ground_truth_facts` to cover every new specific factual claim introduced into kb/about_me.md by Plans 06-03 + 06-04. Optionally add 1-2 voice samples to kb/voice.md if the transcript from Plan 06-01 surfaced quote-worthy Joe-language not yet represented.

**Why this plan is critical (D-A-05 + D-E-01):** Without the ground_truth_facts expansion, Plan 06-06 cat1 judge mis-grades Sonnet's correct citations of the new about_me.md content as fabrication — the EXACT same bug class that STATE.md describes for Plan 05-12 Task 0 iter-2 expansion (cat1-fab-014, cat1-fab-015, cat1-fab-010 all received ground_truth expansions for the same reason). The judge is per-case-isolated; it can only ground claims against the facts list inside the case it's judging.

**Strategy (planner-locked):** Default to expanding ground_truth_facts within EXISTING cases rather than adding new cases. This keeps the D-B-01 hard gate at 15/15 — a known cleanly-passing threshold — rather than perturbing the case count and pass criterion in the same commit as content changes.

The mapping logic: for each new claim X in kb/about_me.md, find the EXISTING cat1 case whose prompt is most likely to surface that claim, and add X to that case's `ground_truth_facts:` list. Multiple claims can map to the same case. The 15 existing cases cover the major fabrication-trap surfaces (counterfactual-project, counterfactual-quantity, persona-expert, compound-fabrication, verifiable-easy) so any new claim should fit.

Output: updated evals/cat-01-fabrication.yaml (still 15 cases) + optionally updated kb/voice.md + 06-05-FACTS-EXPANSION.md (audit trail).
</objective>
