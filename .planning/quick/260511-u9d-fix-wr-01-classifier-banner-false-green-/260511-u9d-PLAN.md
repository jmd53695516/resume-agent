<objective>
Fix WR-01 — the classifier banner false-greens during Anthropic outages because `classifyUserMessage` swallows errors and returns a fail-closed sentinel `{label:'offtopic', confidence:1.0}` instead of throwing. The heartbeat route's `try/catch` therefore never trips and `heartbeat:classifier` gets refreshed to OK even when Haiku is down.

**Approach: Option (a) — extract a throwing variant.**

Add `classifyUserMessageOrThrow` to `src/lib/classifier.ts`. The throwing variant runs the Anthropic call + JSON parse + zod validate and lets errors propagate. The existing `classifyUserMessage` is reduced to a thin wrapper that calls the throwing variant inside its existing try/catch.

The heartbeat route imports the throwing variant; its existing try/catch now actually fires on outage. We also remove the fragile sentinel-shape inspection (`looksFailClosed`) from the heartbeat route — that branch was a workaround for the swallowed-error bug and is no longer needed (and is fragile because a legitimate `'offtopic' + 1.0` Haiku response would false-flag the banner).

**Why option (a) over (b):**
- Intent is explicit at the call site: heartbeat *wants* errors to propagate; chat route *wants* fail-closed deflection.
- The sentinel shape `{label:'offtopic',confidence:1.0}` stops carrying two meanings ("classifier said offtopic with full confidence" vs "classifier crashed"). Joe's heartbeat route had to change the probe phrase from `'health check ping'` to `"Tell me about Joe's PM experience"` precisely because the former was a plausible legitimate offtopic — the same brittleness still exists with the current code.
- Refactor is mechanical: same `try { ... } catch` body, just split across two exported functions.

**Out of scope:**
- The chat route's `classifyUserMessage` call MUST remain fail-closed (don't break the chat hot path).
- No new Anthropic SDK call patterns — reuse the existing `anthropicClient()` + `MODELS.CLASSIFIER`.
- No probe-phrase change in the heartbeat route (already correct).

Purpose: Banner truthfulness — recruiter-facing /api/health must say `classifier=degraded` when classifier is actually degraded.
Output: Two-function classifier API + heartbeat route consuming the throwing variant + regression tests covering both the throw path and the heartbeat banner-truth contract.
</objective>
