This was the longest task by wall-clock. Three sub-attempts:

**Attempt 1 (commit `1d51f00` only):** runId `jRRl8OdDwLXvIL5M6qOjj` @ 2026-05-09T23:50Z — all 15 cases errored with `output_config.format.schema: For 'integer' type, properties maximum, minimum are not supported.` `@ai-sdk/anthropic` translates `z.number().int().min(1).max(5)` to JSON Schema with `minimum`/`maximum` keywords, which Anthropic's tool-input-schema validator rejects on integer types.

**Attempt 2 (`b278b78` — drop user-specified `.min(1).max(5)`):** Same errors — Zod v4's `z.toJSONSchema(z.number().int())` auto-injects `{minimum: -9007199254740991, maximum: 9007199254740991}` (JS safe-int bounds), which Anthropic also rejects.

**Attempt 3 (`2e6e43b` — drop `.int()` entirely):** Cleared the schema-validator path. 1-5 Likert contract stays in prompt copy; out-of-range or fractional scores are warn-loggable downstream but still grade-able. Trade-off: marginally weaker runtime type signal for a working call.
