returns 0/15, every case errors `callAgent failed: 404 Not Found body={"error":"Session unknown"}`).

callAgent invocation. cat6 is untouched (it spawns Playwright; no /api/chat
calls).

gates Phase 5 promotion depends on.

<design_decisions>
1. **One session per CATEGORY, not per RUN, not per CASE.** Per-case (current
   broken state) creates 42+ synthetic ids. Per-run shares one session across
   all 5 cats — which would cumulatively hit the 30-turn cap (CHAT-10 = 60 rows
   = 30 user+assistant pairs). At 15 + 9 + 6 + 5 + 7 = 42 cases, we'd start
   getting `turncap` deflections halfway through. Per-category caps each
   runner's session at its own case count (max 15 for cat1, well under 30) and
   keeps each runner self-contained: no signature change to runCatN, no
   plumbing through scripts/run-evals.ts.

2. **Synthetic email: `eval-cli@joedollinger.dev`.** Joe owns joedollinger.dev
   (already used as RESEND_FROM_EMAIL pattern target). NOT joe.dollinger@gmail.com
   (real recruiter sessions use that — eval rows would pollute admin dashboards
   and trigger Plan 04-04's free-mail abuse classifier). The custom-domain email
   keeps eval rows visually distinct from real traffic in the admin sessions
   table (email_domain='joedollinger.dev').

3. **mintEvalSession lives in agent-client.ts**, not a separate file. It's the
   exact same module already responsible for the /api/chat boundary; sibling
   placement keeps the eval-CLI-to-Next-API contract in one file.

4. **Logger event name `eval_session_minted`** per <constraints>. Implemented
   via childLogger inside mintEvalSession (matches cat-runner pattern).

5. **No turnstile token sent.** /api/session reads NEXT_PUBLIC_TURNSTILE_ENABLED
   at call time; default off in dev. If a future smoke run sets the flag, the
   helper will fail closed with a clear error from /api/session (turnstile_missing
   400) which the caller surfaces. No need to handle turnstile in the helper now;
   doing so would require a server-side test token only Cloudflare issues.

6. **cat6 NOT modified** — verified by reading `src/lib/eval/cats/cat6.ts`: it
   spawns `npx playwright test --grep cat-06`, which hits the live URL through
   browser navigation. The Playwright specs handle their own session minting
   via the EmailGate UI.

7. **No /api/chat or /api/session changes.** Bug is on eval-CLI side only;
   /api/chat's BL-17 404 behavior is correct — eval CLI just needs to mint
   real sessions like the actual chat UI does.
</design_decisions>

    1. Add to `src/lib/eval/agent-client.ts` (alongside existing callAgent / parseChatStream):

    ```typescript
    import { childLogger } from '@/lib/logger';

    const sessionLog = childLogger({ event: 'eval_session_minted' });

    /**
     * Mint a real session_id by POSTing to /api/session, mirroring what the
     * EmailGate UI does. The eval CLI must NOT use synthetic session_id
     * strings — /api/chat (BL-17) validates session existence in Supabase
     * and returns 404 for unknown ids. Use a synthetic but stable email so
     * eval-generated session rows are visually distinct from real recruiter
     * traffic in the admin dashboard.
     *
     * Throws on non-200, missing id field, or network failure — caller's
     * try/catch wraps this into a per-case error rationale (or in the
     * runner-startup case, fails the category with a clear log line).
     *
     * Turnstile NOT supported here. /api/session reads
     * NEXT_PUBLIC_TURNSTILE_ENABLED at call time; default off in dev. If
     * the flag is on during a smoke run, /api/session returns 400
     * turnstile_missing and this helper surfaces that error verbatim.
     */
    export async function mintEvalSession(targetUrl: string): Promise<string> {
      let res: Response;
      try {
        res = await fetch(`${targetUrl}/api/session`, {
          method: 'POST',
          headers: { 'content-type': 'application/json' },
          body: JSON.stringify({ email: 'eval-cli@joedollinger.dev' }),
        });
      } catch (err) {
        throw new Error(
          `mintEvalSession network error for ${targetUrl}/api/session: ${(err as Error).message}`,
        );
      }
      const rawBody = await res.text();
      if (!res.ok) {
        throw new Error(
          `mintEvalSession failed: ${res.status} ${res.statusText} body=${rawBody.slice(0, 200)}`,
        );
      }
      let parsed: unknown;
      try {
        parsed = JSON.parse(rawBody);
      } catch {
        throw new Error(
          `mintEvalSession returned non-JSON body: ${rawBody.slice(0, 200)}`,
        );
      }
      const id = (parsed as { id?: unknown })?.id;
      if (typeof id !== 'string' || id.length === 0) {
        throw new Error(
          `mintEvalSession response missing id field: ${rawBody.slice(0, 200)}`,
        );
      }
      sessionLog.info({ targetUrl, sessionId: id }, 'eval_session_minted');
      return id;
    }
    ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-q00-01 | Spoofing | mintEvalSession synthetic email | accept | `eval-cli@joedollinger.dev` is a domain Joe owns; the address need not exist as a real mailbox because /api/session does not send mail (Plan 04-05 sends mail from /api/chat onFinish, gated by classifier verdict + free-mail check; eval-CLI traffic is always classified as 'normal' for Joe's prompts and the from-domain joedollinger.dev is not free-mail). No spoofing surface beyond what real users have. |
| T-q00-02 | Information disclosure | mintEvalSession logs sessionId | accept | sessionId logged at info level via Pino structured log → Vercel stdout. Same handling as production /api/session inserts (which log via supabaseAdmin error path). Not a secret; nanoid lookup is gated by Supabase service role on /api/chat. |
| T-q00-03 | Denial of service | One eval-CLI run mints up to 5 session rows + ~42 message rows | accept | Each smoke run adds ~5 rows to `sessions` and proportional rows to `messages`. Plan 04-07 archive cron handles cleanup; eval rows are visually identifiable by `email_domain='joedollinger.dev'` for separate retention if needed later. Free-tier Supabase 500 MB cap is 4+ orders of magnitude away. |
| T-q00-04 | Tampering | non-OK /api/session response | mitigate | mintEvalSession throws on non-200 AND on missing-id field AND on non-JSON body. Three independent guards. Each error message includes status code + body prefix for diagnosis. |
| T-q00-05 | Elevation of privilege | eval CLI bypassing turnstile gate | accept | The /api/session route reads NEXT_PUBLIC_TURNSTILE_ENABLED at call time; if a future deploy flips it on, this helper fails closed (the route returns 400 turnstile_missing).
